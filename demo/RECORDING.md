# Recording the demo

Goal: a 45-60 second silent screencast that shows arc-agent-kit moving real USDC on Arc testnet, with English subtitles burned in afterward.

## Before you record

1. **Resize the terminal** to ~110 columns × 32 rows, font 18-20pt, dark theme. The framing matters more than anything else — judges scan thumbnails.
2. **Hide the prompt's git status / right-side info** if your shell shows it. A clean `$` prompt looks more professional.
3. **Test once in dry-run mode** to confirm timing:

   ```bash
   bash demo/run-demo.sh
   ```

   You should see banner → info → balance → simulate → outro, ~35 seconds total.
4. **Make sure your wallet has > 0.001 USDC + a tiny bit of gas-equivalent** (the script sends 0.001 USDC). `npx arc balance <your-address>` to check.

## Recording

macOS has built-in screen recording — no extra software needed.

1. Press **`Cmd` + `Shift` + `5`**. A floating control bar appears at the bottom of the screen.
2. Click the **rectangle-with-dots icon** ("Record Selected Portion"), then drag a tight rectangle around just the terminal window. Leave 8-10 px of padding on each side.
3. Click **`Options`** → uncheck "Show Mouse Clicks" (your script does no clicking; the cursor in the terminal is enough).
4. Click **`Record`**. The control bar collapses.
5. In the terminal, immediately run:

   ```bash
   bash demo/run-demo.sh --real
   ```

   Don't touch anything until the outro banner has been on screen ~4 seconds.
6. Click the **stop icon** in the menu bar (top right). The recording lands on your Desktop as `Screen Recording <date>.mov`.

If it's too long or too short, just re-record. Each `--real` run sends another 0.001 USDC (~$0.0014 on testnet) — cheap.

## Trimming and adding subtitles

The raw `.mov` is fine as-is — you can upload it straight to the hackathon submission form. To make it polished:

### Quick option: iMovie (free, ships with macOS)

1. Open iMovie → New Project → Movie.
2. Drag the `.mov` into the timeline.
3. Trim head/tail dead time with the playhead + `Cmd+B` (split) and Delete.
4. **For subtitles**: select the clip → top-right `T` (titles) → choose "Lower" or "Standard Lower Third" → type the line from `subtitles.srt` → drag to the timecode in the timeline. Repeat for each of the 7 lines in `subtitles.srt`.
5. File → Share → File → Resolution 1080p → Quality High → Save to `demo/demo.mp4`.

### Faster option: hardcode SRT with ffmpeg

If you have `ffmpeg` installed (or can `brew install ffmpeg`):

```bash
ffmpeg -i ~/Desktop/Screen\ Recording*.mov \
  -vf "subtitles=demo/subtitles.srt:force_style='Fontsize=20,PrimaryColour=&Hffffff&,BackColour=&H80000000&,BorderStyle=4,Alignment=2,MarginV=40'" \
  -c:a copy demo/demo.mp4
```

This burns the SRT into the video. ~30 seconds to encode at 1080p.

## What to do with the final file

- **Hackathon submission form**: drag `demo/demo.mp4` into the video field (most platforms cap at ~100 MB, the file should be 20-40 MB).
- **README**: upload as a GitHub Release asset (Releases → Draft new release → drag mp4 in → Publish), then embed the asset URL in `README.md` as a `<video>` tag. GitHub renders mp4 release assets inline.
- **X / Twitter post**: trim to ≤2:20 (the cap), keep the original aspect ratio. The 45-second version is already under that.

## Subtitle timing

The timecodes in `subtitles.srt` are aligned to the `--real` recording. If your run is faster/slower (network latency on the testnet RPC varies by a few seconds), shift the timestamps once in iMovie or pass `-itsoffset` to ffmpeg.

## Reproducibility

The exact same demo can be regenerated by anyone who clones the repo and runs the script. That's the point — `demo/run-demo.sh` is committed, so the demo is part of the repo, not a one-off artifact.
